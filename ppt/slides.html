WHY SVG ICON AGENT
03 / 10
PROBLEM FRAMING

图标不是一张图

ARaster generation
一次生成

文生图可以快速得到像素结果，但小图标需要可编辑、可检查、可复用，错误也要能定位到结构。

  • 输出难以继续编辑
  • 语义对齐缺少可解释诊断
  • 安全、尺寸、颜色规则不易机检
BAgentic vector workflow
协作修复

本项目把图标生成转化为 SVG 程序化图形任务，让多个 Agent 分工判断、选择、优化和修复。

  • 输出为 256×256 可编辑 SVG
  • 候选、批评、选择理由全留痕
  • 本地工具检查安全和可渲染性

AGENT ARCHITECTURE
05 / 10
SYSTEM DESIGN

不是一条链，而是一支小队

LLM-backed Agent 负责目标、规划、生成、批评、选择、优化、验证和修复。本地工具只做可验证的 SVG 语法、安全、渲染和导出。

Memory Tool检索历史经验相似 prompt 的成功/失败模式
Goal Manager生成标准目标、约束、验收条件
Rewriter / Planner压缩意图可执行的 SVG 计划
Generator多候选生成每轮生成 3 个不同 SVG 草稿
Semantic Critic看懂没有识别度、小尺寸语义
SVG Critic能不能用可编辑、安全、渲染风险
Selector共识选择给出 winner 与 repair brief
Optimizer合成反馈批评 + 工具 + 人工建议
ValidatorLLM 验收语义、视觉、规则合规
Taxonomy失败归因语义、布局、安全、复杂度
Router / Refiner路由修复选择重构、简化、局部 patch
Exporter交付物SVG / PNG / gallery / trace
INNOVATION POINTS
06 / 10
ORIGINALITY

Agent 特色集中在六个机制

01
目标管理

把主观 prompt 变成显式 objective、constraints 和 acceptance criteria。

02
检索记忆

从本地历史运行中提取成功模式、失败模式和用户反馈。

03
多候选协作

不是单次采样，生成多个可比较 SVG 草稿并留下评分证据。

04
双 Critic

语义 Critic 看是否像，SVG 质量 Critic 看是否安全、可编辑。

05
失败路由

先分类根因，再决定 safety rebuild、semantic recomposition 或 minor patch。

06
人在环路

Web UI 支持后验 optimizer feedback，不必重跑完整规划链。
